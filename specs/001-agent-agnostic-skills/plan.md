# Implementation Plan: Agent-Agnostic Skills Framework

**Input**: Feature specification from `/specs/001-agent-agnostic-skills/spec.md`

## Summary

This feature implements a framework for creating and deploying agent-agnostic skills, tools, and subagents. It modularizes and fosters the integration of:
1.  **Skills (NL)**: Markdown procedures with OpenCode Standard Frontmatter.
2.  **Tools (Executable)**: MCP-compliant servers for deterministic capabilities.
3.  **Scripts (Python/Shell)**: Exposed as tools via an MCP-compliant runner.
4.  **Subagents**: Markdown definitions that compose skills and tools into high-level agent logic.

A unified `skills.json` manifest file acts as the source of truth, and a Python-based deployment tool synchronizes these artifacts into harness-specific configuration directories (Gemini CLI, Claude Code, OpenCode).

## Technical Context

**Language/Version**: Python 3.11+, Bash/Shell
**Primary Dependencies**: `mcp` (SDK), `pydantic`, `pyyaml`

**Performance Goals**: Discovery latency < 100ms; multi-agent sync latency < 5s
**Constraints**: Must maintain compatibility with OpenCode frontmatter and MCP protocol
**Scale/Scope**: Support for 100+ skills, tools, and subagents in a single repository
